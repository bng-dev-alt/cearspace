## 2. AI Workflow & JSON struktura

### Kontext (Context)
Při generování se odesílá název a popis projektu, stávající rozložení sloupců a aktuální seznam úkolů, aby AI nenavrhovala to, co již existuje.

### JSON Výstup od AI
AI vrací strukturovaný JSON:
```json
{
  "projectSummary": "Stručný souhrn projektu a doporučený postup...",
  "tasks": [
    {
      "title": "Název úkolu",
      "description": "Podrobný popis",
      "priority": "High" | "Medium" | "Low",
      "estimate": "4h",
      "acceptanceCriteria": ["kritérium 1"],
      "checklist": ["položka 1"],
      "recommendedColumn": "Todo"
    }
  ]
}
```

## 3. Duplicate Handling (Řešení duplicit)

Pokud generovaný úkol shoduje název (case-insensitive, ořezáno) se stávajícím úkolem na boardu, uživatel je upozorněn varovným badge a má na výběr ze tří akcí pro daný konkrétní úkol:
- **Skip (Přeskočit)** (výchozí) — Úkol se neimportuje.
- **Replace (Nahradit)** — Původní úkol se z boardu odstraní (smaže) a importuje se nová verze.
- **Create Anyway (Vytvořit i tak)** — Úkol se vytvoří jako duplicitní.

Při stisku **Importovat backlog** se provede příslušný import a u každého importovaného úkolu se zaznamená aktivita `Generated by AI`.

## 5. Klávesové zkratky (UX vylepšení)

### Zavření dialogu klávesou ESC
- **Popis**: Zavedli jsme standardní React `useEffect` hook, který naslouchá na globální událost `keydown`. Pokud uživatel stiskne klávesu **`Escape`**, zavře se modal dialog backlog generátoru bez ohledu na jeho aktuální stav (zajišťuje konzistentní chování s ostatními modaly v aplikaci).
- **Testování**: Do integračních testů [`generate-tasks.test.tsx`](file:///Users/beng/Cursor%20-%20Projects/kanban_antigravity/frontend/src/__tests__/generate-tasks.test.tsx) byl přidán samostatný unit test ověřující, že stisk klávesy `Escape` na okně zavolá callback `onClose` a okno se řádně uzavře.
